Test Scope

Introduction

The system to be tested is the VT smart parking system. The system would enable the users to buy various types of Parking permits offered and then show the user the nearest spot available based on the category of the users like students, employees, visitors, etc. and bill the user for the parking.

Requirements Tested

Functional Requirements

  • FR-1. The System should provide a login and logout functionality using the Virginia Tech PID and a user-defined password.
  • FR-2. The System should be GPS-enabled in order to take in the user’s live location.
  • FR-3. The System should suggest available Parking slots using the user’s location in ascending order of distance.
  • FR-4. The System should leverage image processing to capture if a certain Parking slot is available. Based on this availability, the System should update the database and also return the same results to the user
  • FR-5. The System should allow the user to search for the various Parking lots across campus.
  • FR-6. The System should let the user book and pay for the Parking spot using Parking permits.
  • FR-7. In case the user does not have an already existing Parking permit, the System should allow the user to buy a fresh Parking permit.
  • FR-8. The System should enable the user to enter their details.
  • FR-9. The System should display the Parking duration and Parking details.
  • FR-10. The System should allow Parking owners to visualize parking patterns. Assumption: It will be a graph with the parking slot locations and times showing the user trend of the parking availability over the last n days.
  • FR-11(NFR-5). The application must be able to store the Parking information of the student within the past 90 days.

Non-Functional Requirements

  • NFR-1. The Smart Parking application should be able to run both on Android (version 7 and above) and iOS (version 10 and above).

Out of scope requirements of the testing plan

  • NFR-2. Any Student data will be encrypted with the required encryption standards.
  • NFR-3. The application must have an uptime of 99.7 percent.
  • NFR-4. The System shall be capable of recovering from an application or System failure, leaving the application and System in an operable state.
  • NFR-6. The Application should address future requirement changes or handle new requirements.

Test Plan

Test Procedure

Test Goal Procedure
1.1 If a new user, has the option to register Open the application on an android and iOS device, there should be a “register” option which should open a registration page.
1.2 If new user, should register successfully On the registration page, the user should enter the VT PID and set a password and logout and be redirected to the login screen.
1.3 The existing user should have the option to login Open the application on an android and iOS device, there should be a signin option with VT PID and password fields.
1.4 An existing user, should be able to login successfully with valid password Enter the VT PID and password in the login screen and click on the sign in button, the user should be redirected to the home screen.
1.5 A logged-in user should have the option to logout successfully When logged in, after clicking on the Logout button, the user should be able to Logout and redirect to the login screen.
